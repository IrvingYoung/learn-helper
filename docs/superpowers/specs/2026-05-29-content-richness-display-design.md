# 知识来源丰富度与展示友好度优化设计

## 背景

当前 Learn Helper 的知识内容严重不足：
- 种子数据仅 13 个知识点，`content`/`code_examples`/`common_mistakes` 字段全部为空
- 练习题仅 6 道，`solution_detail`/`sample_code`/`common_errors` 为空
- 前端知识点详情页纯白底文字堆叠，缺少视觉层次
- 知识树导航只有文字+展开按钮，无状态/进度信息
- 练习页功能单薄，缺少视觉引导
- 无学习路径指引，用户不知道下一步学什么

## 方案

**AI 批量生成静态种子数据 + 前端展示页面全面优化。**

种子数据一次性生成、版本控制、可重复运行。前端不增加运行时 AI 依赖。

## 二、AI 种子数据生成脚本

### 工具

Python 脚本 `scripts/generate_seeds.py`，调用 Claude API (Haiku)。

### 工作流

1. 脚本内定义知识点树结构（Python dict），含 name、slug、parent_slug、difficulty、sort_order
2. 对每个知识点调用 Claude API 生成：
   - `content`：500-800 字 Markdown（概念讲解 + 核心原理 + 应用场景）
   - `code_examples`：2-3 个代码示例，JSON 格式 `[{lang, code, explanation}]`
   - `common_mistakes`：3-5 个常见错误，JSON 数组
3. 对每个练习题调用 Claude API 生成：
   - `description`：题目描述含示例输入/输出
   - `hints`：3 层逐步提示，JSON 数组
   - `solution_outline`：解题思路概要
   - `solution_detail`：详细解答（Markdown）
   - `sample_code`：示例代码，JSON 格式
   - `common_errors`：常见错误，JSON 数组
4. 输出为 `backend/db/seed/seed.sql`，包含 INSERT 语句

### Prompt 模板

**知识点生成 prompt：**

```
你是一位数据结构与算法的教学专家。请为以下知识点生成详细的教学内容。

知识点名称：{name}
难度级别：{difficulty}
上级知识点：{parent_name}
关键要点：{key_points}

请生成以下内容（JSON 格式）：
{
  "content": "500-800字 Markdown 格式的概念讲解，包含：1.概念定义 2.核心原理 3.应用场景 4.与其他数据结构/算法的对比",
  "code_examples": [{"lang": "Python", "code": "...", "explanation": "..."}],
  "common_mistakes": ["错误1", "错误2", ...]
}

要求：
- content 用中文撰写，代码注释可用中文
- 代码示例 2-3 个，优先 Python，可选 Go
- 常见错误 3-5 个，每个一句话描述
```

**练习题生成 prompt：**

```
你是一位算法面试题设计专家。请为以下知识点设计一道算法练习题。

关联知识点：{topic_name}
难度级别：{difficulty}
题目类型：{type}

请生成以下内容（JSON 格式）：
{
  "title": "题目标题",
  "description": "题目描述，含示例输入/输出，Markdown 格式",
  "hints": ["提示1：思路方向", "提示2：具体方法", "提示3：实现要点"],
  "solution_outline": "解题思路概要",
  "solution_detail": "详细解答，Markdown 格式，含代码",
  "sample_code": [{"lang": "Python", "code": "..."}],
  "common_errors": ["常见错误1", "常见错误2"],
  "time_complexity_expected": "O(...)",
  "space_complexity_expected": "O(...)"
}
```

### 成本控制

- 使用 Claude Haiku 降低成本
- 脚本支持断点续传：已生成的知识点/练习题跳过（基于输出文件判断）
- API 调用间加入 1 秒延迟避免限流
- 输出 JSON 中间文件 + 最终 SQL，便于调试和增量更新

### 输出

- 中间文件：`scripts/output/topics/` 和 `scripts/output/exercises/`（JSON 格式）
- 最终输出：`backend/db/seed/seed.sql`（替换现有文件）

## 三、知识点详情页排版优化

### 当前问题

纯白底文字堆叠，没有视觉层次，内容区域像无格式文档。

### 改进

1. **卡片化分区**：content、code_examples、common_mistakes、key_points 各自放在独立卡片中，带标题图标和圆角边框
2. **难度标签增强**：更醒目的颜色和图标（beginner=绿色 seedling，intermediate=黄色 compass，advanced=红色 flame）
3. **面包屑导航**：详情页顶部显示路径，如 `数据结构与算法 > 基础数据结构 > 数组`
4. **关联练习题卡片**：详情页底部展示关联练习题，每题显示难度标签和完成状态
5. **学习状态标记**：详情页顶部增加状态切换按钮（未开始/学习中/已掌握），带颜色变化

## 四、知识树导航增强

### 当前问题

只有文字 + 展开箭头，缺少进度和状态信息。

### 改进

1. **学习状态图标**：每个节点前显示小圆点（灰色=未开始，黄色=学习中，绿色=已掌握）
2. **进度条**：知识图谱标题下方显示整体进度条（已学/总数）
3. **当前路径高亮**：选中节点的祖先链全部高亮
4. **关联练习题数量**：叶子节点后显示关联题目数，如 `数组 (3题)`

## 五、练习页视觉增强

### 当前问题

功能单薄，缺少视觉引导。

### 改进

1. **难度颜色标签**：easy=绿色，medium=黄色，hard=红色，带图标
2. **知识点标签**：每道题显示关联知识点标签（小 chip）
3. **完成状态图标**：未开始=空心圆，进行中=半填充，已完成=实心对勾
4. **筛选器优化**：筛选条件用 pill/chip 形式，已选条件可快速清除
5. **题目卡片**：每道题用卡片展示，含标题、难度、标签、状态，hover 有阴影

## 六、学习路径引导

### 当前问题

用户不知道下一步该学什么。

### 改进

1. **前置/后续知识点**：知识点详情页显示"前置知识"和"下一步学习"链接，基于树结构计算（父节点的前一个兄弟=前置，当前节点的下一个兄弟/子节点=后续）
2. **推荐下一步**：详情页底部，显示下一个应该学习的知识点卡片
3. **空状态引导**：首次进入 `/learn` 时，显示引导提示"建议从'数组'开始学习"

### 引导示意

```
┌──────────────────────────────────────────────────┐
│ ← 上一步                           下一步 →     │
│ 链表                               二分查找      │
│ (前置知识)                         (推荐学习)    │
└──────────────────────────────────────────────────┘
```

## 实现范围

### 数据层

- 新增 `scripts/generate_seeds.py`：AI 种子数据生成脚本
- 替换 `backend/db/seed/seed.sql`：扩展知识点树 + 丰富内容 + 增加练习题
- 数据库 schema 无变更（现有字段已预留 content/code_examples/common_mistakes 等）

### 后端

- `GET /api/topics` 返回数据需包含练习题数量和完成状态（需 JOIN learning_records）
- `GET /api/topics/:slug` 返回数据需包含前置/后续知识点信息
- `GET /api/exercises` 返回数据需包含关联知识点名称
- 无 schema 变更，仅调整查询和返回结构

### 前端

- 重构 `LearnPage`：知识树导航增强（状态图标、进度条、路径高亮、题目数量）
- 重构知识点详情区域：卡片化排版、面包屑、学习状态标记、关联练习卡片
- 新增学习路径引导组件：前置/后续知识点、推荐下一步、空状态引导
- 重构 `PracticePage`：题目卡片、难度/知识点标签、完成状态图标、筛选器优化
- 新增 `/learn/:slug` 路由页面（当前知识点详情在 LearnPage 内部渲染，需拆出独立页面以支持 URL 直达）

### 不包含

- 数据库 schema 变更
- 新的 API 端点
- 运行时 AI 内容生成
- 学习规划角色或仪表盘改动
