# PWA Support Design

## Scope

Add Progressive Web App support to LLM Wiki: installable to home screen, works offline with cached app shell and previously-viewed wiki pages.

## Core Decisions

1. **Scope**: Install + offline shell + cached pages (no offline write queue, no push notifications)
2. **Update strategy**: Background silent (`registerType: 'autoUpdate'`)
3. **Install prompt**: None — rely on browser native install affordances
4. **Icons**: Generated from existing brand SVG via sharp
5. **Implementation**: `vite-plugin-pwa` (Workbox under the hood)

## Section 2: Vite Plugin & Service Worker Strategy

**New dependency**: `vite-plugin-pwa`

**`vite.config.ts` additions**:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['favicon.svg', 'icons/*.png'],
      manifest: {
        name: 'LLM Wiki — 个人知识库',
        short_name: 'LLM Wiki',
        description: 'AI 维护的个人知识库',
        theme_color: '#c45c26',
        background_color: '#fbf8f3',
        display: 'standalone',
        start_url: '/wiki',
        scope: '/',
        icons: [
          { src: '/icons/icon-192.png', sizes: '192x192', type: 'image/png' },
          { src: '/icons/icon-512.png', sizes: '512x512', type: 'image/png' },
          { src: '/icons/icon-maskable-512.png', sizes: '512x512', type: 'image/png', purpose: 'maskable' },
        ],
      },
      workbox: {
        runtimeCaching: [
          {
            urlPattern: /^\/api\/wiki\/[^/]+$/,
            handler: 'StaleWhileRevalidate',
            options: {
              cacheName: 'wiki-pages',
              expiration: { maxEntries: 50, maxAgeSeconds: 60 * 60 * 24 * 7 },
            },
          },
          {
            urlPattern: /^\/api\/wiki\/tree/,
            handler: 'StaleWhileRevalidate',
            options: { cacheName: 'wiki-tree' },
          },
        ],
        navigateFallback: '/index.html',
      },
    }),
  ],
  // ... existing server config
})
```

**`src/main.tsx` SW registration**:

```ts
import { registerSW } from 'virtual:pwa-register'
registerSW({ immediate: true })
```

**Offline behavior**:
- App shell (HTML/JS/CSS) always accessible (precache)
- Previously viewed wiki pages: readable offline (SWR fetches from SW cache)
- New pages / write operations / AI chat: fail, handled by existing SWR error UI

**Update flow**:
- Plugin checks for SW updates on load
- New version downloaded in background
- User gets new version on next navigation (no prompt)

## Section 3: Icon Generation Script

**New file**: `frontend/scripts/generate-icons.mjs`

```js
import sharp from 'sharp'
import { readFileSync, mkdirSync } from 'fs'
import { join, dirname } from 'path'
import { fileURLToPath } from 'url'

const __dirname = dirname(fileURLToPath(import.meta.url))
const src = join(__dirname, '..', 'public', 'favicon-source.svg')
const outDir = join(__dirname, '..', 'public', 'icons')
mkdirSync(outDir, { recursive: true })

const svg = readFileSync(src)

const sizes = [
  { size: 192, name: 'icon-192.png' },
  { size: 512, name: 'icon-512.png' },
]

for (const { size, name } of sizes) {
  await sharp(svg).resize(size, size).png().toFile(join(outDir, name))
  console.log(`Generated ${name}`)
}

const maskableSvg = `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" fill="#c45c26"/>
  <g transform="translate(16 16) scale(0.4) translate(-16 -16)">
    <path d="M16 8v16M8 16h16" stroke="white" stroke-width="6" stroke-linecap="round" fill="none"/>
  </g>
</svg>`
await sharp(Buffer.from(maskableSvg)).resize(512, 512).png()
  .toFile(join(outDir, 'icon-maskable-512.png'))
console.log('Generated icon-maskable-512.png')
```

**`package.json` additions**:

```json
"scripts": {
  "icons": "node scripts/generate-icons.mjs"
}
```

**Build flow changes**:
- First time: run `npm run icons` to generate PNGs (results committed to git)
- Subsequent: `npm run build` auto-generates `dist/sw.js` and `dist/manifest.webmanifest`
- All of `dist/**` is precached

## Testing

After `npm run build && npm run preview`:

1. **Service Worker active**: Chrome DevTools → Application → Service Workers → confirm SW is activated
2. **Manifest valid**: Application → Manifest → all fields correct
3. **Install prompt**: Chrome address bar should show install icon (or check `beforeinstallprompt` event in console)
4. **Offline test**:
   - Network → Offline mode
   - Reload page → app shell loads
   - Navigate to a previously-visited page → content shows from cache
   - Navigate to a new page / send chat message → fails (existing SWR error UI)
5. **Lighthouse**: PWA category should score 100
6. **Mobile simulator**: DevTools device mode → "Add to home screen" should work

---

## What Stays Unchanged

- All existing React components, routes, state management
- Backend — zero backend changes
- Existing favicon SVG (extracted to file, same data)
- Build output structure (just adds `sw.js` and `manifest.webmanifest` to `dist/`)
