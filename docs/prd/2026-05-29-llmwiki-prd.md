## 1、项目背景

### 1.1 核心概念

**LLM Wiki** — 一个由 AI Agent 自动维护的个人知识库。

产品由两层构成：

```
你 (用户) ← 对话 → AI Agent ← 读/写 → LLM Wiki (SQLite)
                                        ↑
                            原始资料（临时读取，不存储）
```

- **LLM Wiki**：数据层，一棵由 wiki_pages 组成的知识树。每个节点是一篇页面，页面内容就是知识本身
- **AI Agent**：逻辑层，通过对话理解你的意图，读/写/维护 Wiki。它是你知识库的"管理员"
- **你**：通过对话提出要求、做决策；通过页面浏览器阅读知识
- **原始资料**：你扔给 AI 的文章、文件，AI 读完吸收后不保存原文，只保留蒸馏后的知识

### 1.2 与 Karpathy LLM Wiki 的关系

| 维度 | Karpathy 原版 | 本产品 |
| --- | --- | --- |
| 存储 | 原始资料 + Wiki 页面（文件系统） | ✅ 只存 Wiki 页面（SQLite） |
| 原始资料 | 只读，永久保存 | ⚡ 临时读取，吸收后不存 |
| 交互 | 终端 + Obsidian | ✅ Web UI（三栏布局） |
| 写入流程 | 直接写 | ✅ 预览 → 确认 → 写入 |
| 概览页 | index.md | ✅ "概览页面"，LLM 自动维护 |

### 1.3 设计原则

- **所有写入需确认**：AI 预览 → 你确认 → 才写入
- **纯对话操作**：没有按钮、表单、拖拽，所有操作通过聊天完成
- **概览页面自动维护**：LLM 每次变更后自动更新

## 2、产品定位

一句话：**告诉 AI 你想学什么，AI 帮你建知识库。**

适用任何领域（编程、摄影、历史、咖啡……），不限范围。

## 3、用户角色

单用户，无登录，所有功能对用户完全开放。

## 4、核心用例

### 4.1 通用确认流程

所有写入操作（创建、修改、删除、移动页面）都必须经过：

```
AI 生成预览 → 聊天展示 → 用户反馈调整 → 用户确认 → 写入
```

知识树只反映已确认写入的状态。确认前的变更只在聊天中展示。

### 4.2 用例

| 编号 | 用例 | 操作入口 | 流程 |
| --- | --- | --- | --- |
| UC1 | 新建知识领域 | 聊天输入："我要学 Go 后端" | AI 生成大纲树 → 对话调整 → 确认 → 创建页面 |
| UC2 | 展开页面内容 | 聊天输入："展开讲讲 goroutine" | AI 写内容预览 → 确认 → 写入页面 |
| UC3 | 吸收资料 | 粘贴文章/拖拽文件到聊天 | AI 阅读 → 分析 → 展示变更预览(新增/更新哪些页面) → 确认 → 写入 |
| UC4 | 对话沉淀 | 讨论中或结束后 "保存到知识库" | AI 蒸馏对话 → 展示摘要 → 确认 → 写入 |
| UC5 | 按需生成 | 聊天输入："总结一下 Redis" / "出几道题" | AI 生成 → 展示 → 确认 → 入库 |
| UC6 | 浏览/复习 | 点击左栏知识树节点 | 右栏展示页面内容，纯读取 |
| UC7 | 查看进度 | 聊天输入："我学得怎么样了" | AI 查看概览页面 + 分析覆盖情况 → 聊天展示 |

### 4.3 吸收资料详解

```
用户：粘贴一篇文章 或 拖拽一个文件
       ↓
AI 读取并分析文章内容
       ↓
AI 在聊天中回复：
  "我看完了，这篇文章主要讲 goroutine 的 GMP 调度模型。
   我建议：
   • 在 goroutine 页面补充'调度原理'章节
   • 新增一个子页面 'GMP 模型'
   确认吗？"
       ↓
用户：可以 / 调整 / 拒绝
       ↓
确认后：AI 更新 wiki_pages → 更新概览页面 → 左栏/右栏同步
```

## 5、数据模型

### 5.1 wiki_pages

知识库的唯一载体。一棵树，每个节点是一篇页面。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 自增主键 |
| title | TEXT | 页面标题，如 "goroutine" |
| slug | TEXT UNIQUE | URL 标识 |
| page_type | TEXT | entity / concept / overview / ... |
| content | TEXT | 页面内容（Markdown） |
| tags | JSON | 标签列表 |
| parent_id | INTEGER FK | 父页面，自引用树结构 |
| content_status | TEXT | empty / draft / published |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.2 概览页面

一个特殊的 wiki_page（page_type = "overview"），由 AI Agent **自动维护**，不通过确认流程（因为 AI 只是总结已有知识，不引入新信息）。

包含：
- 知识树总览（顶级领域 + 页面数）
- 最近更新的页面
- 空页面列表（大纲已建但内容未填充）
- 总页面数和内容覆盖情况

用户打开知识库时默认展示。

### 5.3 与现有表的关系

- **topics 表**：保留。已有的知识点作为第一棵知识树，待迁移到 wiki_pages
- **exercises / learning_records**：废弃，不维护
- **conversations / messages**：保留，AI 对话历史继续使用
- **ai_configs**：保留，AI 模型配置继续使用

## 6、UI 交互设计

### 6.1 布局

三栏可折叠：

| 栏 | 内容 | 可折叠 | 可拖拽 |
| --- | --- | --- | --- |
| 左栏 | 知识树（Wiki 完整索引） | ✅ | ✅ |
| 中栏 | AI 聊天（唯一操作入口） | — | ✅ |
| 右栏 | 页面浏览器 | ✅ | ✅ |

### 6.2 知识树节点状态

| 颜色 | 含义 |
| --- | --- |
| 绿色 | 内容已填充 |
| 黄色 | 部分填充 |
| 灰色 | 空页面 |
| 蓝色 | 当前选中 |

### 6.3 顶部导航

```
[Learn Helper]   🧠 知识库 | 📖 浏览 | 📊 仪表盘
```

### 6.4 AI 交互规则

- 聊天是唯一操作入口（无按钮、表单、拖拽）
- 查知识库：聊天中手动开关，默认关闭
- 所有写入操作必须经过确认
- 概览页面除外（纯总结已有知识，自动维护）

## 7、技术方案

### 7.1 架构

```
┌──────────────────────────────────────────┐
│         前端 (React + Vite)              │
│   三栏布局 | 聊天面板 | Markdown 渲染    │
└─────────────────┬────────────────────────┘
                  │ REST API + SSE
┌─────────────────▼────────────────────────┐
│        Go 后端 (Chi + SQLite)            │
│                                          │
│   ┌────────────┐  ┌──────────────────┐   │
│   │ Wiki 模块  │  │ AI 模块          │   │
│   │ (CRUD 页面)│  │ (Provider 抽象)   │   │
│   └──────┬─────┘  └────────┬─────────┘   │
│          └─────────────────┘              │
│                      ▼                    │
│              ┌──────────────┐             │
│              │   SQLite     │             │
│              │  (wiki_pages │             │
│              │   / topics)  │             │
│              └──────────────┘             │
└──────────────────────────────────────────┘
```

### 7.2 API

```
GET    /api/wiki           — 获取完整知识树
GET    /api/wiki/:slug     — 获取页面内容
POST   /api/wiki           — 创建页面（AI Agent 内部调用）
PUT    /api/wiki/:id       — 更新页面（AI Agent 内部调用）
DELETE /api/wiki/:id       — 删除页面（AI Agent 内部调用）
```

POST / PUT / DELETE 由 AI Agent 在用户确认后调用，不对外暴露。

### 7.3 AI Provider

复用现有抽象层（Claude + DeepSeek），新增 Wiki 维护角色的 System Prompt。

### 7.4 概览页面维护策略

每次写入操作完成后，AI Agent 自动更新概览页面（不经过确认流程），包括：
- 页面数和覆盖领域
- 最近变更摘要
- 空页面列表

## 8、第一期范围

### 8.1 包含

- 三栏布局 UI（可折叠、可拖拽）
- AI 聊天创建/更新知识树
- 粘贴/上传文件吸收知识（.md / .txt / .pdf，≤ 5MB）
- 确认流程（预览 → 确认 → 写入）
- 概览页面自动维护
- 现有的 topics 保留为第一棵知识树

### 8.2 不包含

- 全文搜索
- 数据导出
- 文件夹同步
- 练习/学习模式

## 9、风险

| 风险 | 应对 |
| --- | --- |
| AI 输出质量不稳定 | 所有写入需确认；system prompt 约束 |
| AI API 不可用 | 浏览已有 Wiki 不受影响 |
| 文件超出 5MB | 聊天提示压缩后重试 |
| AI 分析失败 | 聊天提示重试或换文件 |

### 6.5 结构操作 vs 知识操作

| 操作类型 | 示例 | 需要确认？ | 操作方式 |
| --- | --- | --- | --- |
| 结构操作 | 新建空页面、改名、移动、排序、删空页面 | ❌ 直接执行 | 知识树上直接操作 |
| 知识操作 | AI 写/改内容、删有内容的页面、合并/拆分 | ✅ 预览→确认 | 聊天操作 |

**结构操作不经过确认，因为空页面没有信息价值。** 知识树上直接：

- 点击"+"新建子页面 → 输入标题 → 立即创建
- 拖拽节点 → 移动位置
- 右键 → 改名 / 删除
- 以上操作 AI 同步更新概览页面

只有涉及知识内容本身的操作，才走确认流程。
