# Spec: page-name-search

用户在知识库树中按页面名称快速搜索定位

## Overview

在知识树侧边栏增加一个搜索输入框。用户输入关键词时，前端实时过滤知识树节点，仅保留 title 包含关键词的节点及其祖先路径。这是纯前端功能，不涉及后端改动。

## Behavior

### Requirement: 知识树顶部显示搜索输入框

搜索输入框固定在知识树组件顶部，在"知识库"标题下方、树节点列表上方。

#### Scenario: 搜索框始终可见
- **WHEN** 知识树侧边栏展开时
- **THEN** 树顶部显示一个搜索输入框，带有搜索图标和占位文本

#### Scenario: 清空按钮
- **WHEN** 搜索框有内容时
- **THEN** 输入框右侧显示清空按钮（×）
- **AND** 点击清空按钮后搜索框清空，树恢复完整展示

### Requirement: 按页面名称过滤树节点

系统根据搜索关键词过滤树节点，只显示 title 包含关键词的节点。

#### Scenario: 搜索匹配
- **WHEN** 用户在搜索框输入关键词
- **THEN** 树中仅显示 title 包含该关键词（大小写不敏感）的节点
- **AND** 匹配节点的所有祖先节点保持可见（提供路径上下文）
- **AND** 不匹配的节点及其子树隐藏

#### Scenario: 搜索无匹配
- **WHEN** 用户输入的关键词没有匹配任何页面名称
- **THEN** 树显示为空状态提示

#### Scenario: 清空搜索恢复
- **WHEN** 用户清空搜索框内容
- **THEN** 树恢复显示完整节点

### Requirement: 搜索时自动展开祖先节点

搜索状态下，匹配节点的所有祖先节点自动展开，确保用户能看到匹配节点在树中的位置。

#### Scenario: 搜索时展开
- **WHEN** 搜索结果包含匹配节点
- **THEN** 所有匹配节点的祖先节点自动展开
- **AND** 清空搜索后恢复用户的原始展开/折叠状态

### Requirement: 搜索状态下禁用编辑操作

搜索状态下，拖拽、右键菜单、双击重命名等操作功能应禁用，避免在过滤视图中产生不一致的操作体验。

#### Scenario: 搜索时禁用拖拽
- **WHEN** 搜索框有内容时
- **THEN** 树节点不可拖拽
