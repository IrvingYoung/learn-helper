GMP 模型

已发布 最后更新: 2 小时前
并发 调度 Go

概述

GMP 是 Go 语言并发模型的核心机制，代表 Goroutine（G）、Machine（M）和 Processor（P）三个关键概念。理解 GMP 模型是掌握 Go 并发编程的基础。

核心概念

Goroutine (G)

Goroutine 是 Go 语言中的轻量级线程，由 Go 运行时管理。相比操作系统线程，Goroutine 的创建和切换成本极低，初始栈大小仅 2KB。

Machine (M)

M 代表操作系统线程，是执行 Goroutine 的实体。Go 运行时会维护一个 M 的线程池，但 M 的数量通常远少于 Goroutine 的数量。

Processor (P)

P 是逻辑处理器，代表执行 Go 代码所需的上下文。P 的数量由 GOMAXPROCS 决定，默认等于 CPU 核心数。

调度流程

  1. 创建 Goroutine 时，G 被放入 P 的本地队列
  2. P 的本地队列满时，G 被放入全局队列
  3. M 从 P 的本地队列获取 G 执行
  4. 本地队列空时，M 尝试从全局队列或其他 P 偷取 G
  5. G 阻塞时，M 与 P 分离，P 分配给其他 M

关键要点

M:N 调度 工作窃取 协程调度
